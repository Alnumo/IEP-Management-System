## 📋 Translation Standards & Principles

### 1. Medical & Therapy Terminology Accuracy

**Core Principle**: Medical accuracy is paramount. Always use established medical translations.

#### Key Guidelines:
- **Consistency First**: Once a medical term is translated, use the same translation throughout
- **Professional Standards**: Follow WHO Arabic medical terminology standards
- **Local Context**: Adapt to Saudi Arabian medical practices and regulations
- **Clarity Over Literal**: Prioritize clear understanding over word-for-word translation

#### Example Standards:
```
✅ Correct:
Individual Education Plan → خطة التعليم الفردية
Therapy Session → جلسة علاجية
Assessment → تقييم
Behavioral Intervention → تدخل سلوكي

❌ Avoid:
Individual Education Plan → خطة التعليم الشخصية (too personal)
Therapy Session → جلسة معالجة (incorrect medical term)
Assessment → تقدير (too general)
```

### 2. Cultural Sensitivity Guidelines

#### Religious & Cultural Considerations:
- Use "إن شاء الله" for future planning where culturally appropriate
- Respect Islamic calendar references alongside Gregorian dates
- Gender-appropriate language for mixed or specific gender contexts
- Respect for family hierarchy in communications

#### Professional Courtesy:
```
Arabic Opening: أهلاً وسهلاً بكم في مركز نمو أركان
English Opening: Welcome to Arkan Growth Center

Arabic Closing: بارك الله فيكم
English Closing: Thank you for choosing Arkan
```

### 3. Gender-Neutral Language Considerations

#### Arabic Challenges & Solutions:
- Use dual forms where possible: الطلاب والطالبات
- Employ professional titles: المختص/ة, الطبيب/ة
- Alternative neutral constructions: الشخص المعني, الجهة المختصة

```
✅ Gender-Inclusive:
المختص في العلاج (The therapy specialist)
فريق الرعاية (Care team)
ولي الأمر (Guardian - naturally neutral)

❌ Avoid Gender-Specific:
المعالج (masculine only)
الممرضة (feminine only)
```

## 🎯 RTL-Specific Considerations

### 1. Text Direction Impact on UI Copy

#### Navigation & Flow Indicators
```
LTR English: "Next →"
RTL Arabic: "← التالي"

LTR English: "Back | Home | Forward"
RTL Arabic: "للأمام | الرئيسية | رجوع"
```

#### Breadcrumb Navigation
```

RTL: الجلسات < أحمد < الطلاب < الرئيسية
```

### 2. Number Formatting

#### Arabic-Indic vs Western Numerals
| Context | Arabic UI | English UI | Notes |
|---------|-----------|-------------|-------|
| Phone Numbers | ٠١٢٣٤٥٦٧٨٩ | 0123456789 | User preference |
| Dates | ٢٠٢٤/١٢/٢٥ | 25/12/2024 | Consider Hijri calendar |
| Currency | ١٢٣.٤٥ ر.س | 123.45 SAR | Saudi Riyal formatting |
| Percentages | ٨٥٪ | 85% | Statistical data |
| Session Count | الجلسة رقم ٥ | Session #5 | Progress tracking |

#### Implementation Notes:
```typescript
// Number formatting utility
const formatNumber = (num: number, locale: 'ar' | 'en'): string => {
  if (locale === 'ar') {
    return num.toLocaleString('ar-SA');
  }
  return num.toLocaleString('en-US');
};
```

## ✅ Quality Assurance Guidelines

### Review Checklist for Translators

#### 1. Medical Accuracy ✓
- [ ] All medical terms use standard WHO Arabic translations
- [ ] Therapy-specific terminology is consistent throughout
- [ ] Saudi medical regulations terminology is respected
- [ ] No ambiguous medical language

#### 2. Cultural Appropriateness ✓
- [ ] Islamic cultural values respected
- [ ] Saudi social norms considered
- [ ] Gender-appropriate language used
- [ ] Professional courtesy maintained

#### 3. Technical Consistency ✓
- [ ] UI element translations are uniform
- [ ] Error messages are clear and actionable
- [ ] System terminology is standardized
- [ ] i18n keys follow naming conventions

#### 4. Linguistic Quality ✓
- [ ] Grammar is correct in both languages
- [ ] Vocabulary is appropriate for target audience
- [ ] Tone is professional and welcoming
- [ ] Register is consistent (formal/informal)

### Common Mistakes to Avoid

#### ❌ Translation Errors
1. **Literal Translation**: "Save" → "احفظ" (correct: "حفظ")
2. **Gender Inconsistency**: Mixing masculine/feminine forms
3. **Register Mismatch**: Using informal Arabic in professional context
4. **Cultural Insensitivity**: Ignoring Islamic values in messaging

#### ❌ Technical Errors
1. **Inconsistent Terms**: Using different Arabic terms for same English word
2. **Missing Context**: Translating without understanding feature purpose
3. **UI Breaking**: Arabic text too long for UI containers
4. **Number Format**: Using wrong numeral system for context

#### ❌ Medical Errors
1. **Incorrect Terminology**: Using colloquial terms for medical concepts
2. **Ambiguous Meaning**: Creating confusion in medical instructions
3. **Legal Issues**: Mistranslating consent or liability terms

```
domain.category.element.attribute
```

#### Examples
```typescript
// Medical domain
i18n.t('medical.therapy.session.title') // "جلسة علاجية"
i18n.t('medical.assessment.comprehensive') // "تقييم شامل"
i18n.t('medical.diagnosis.autism') // "اضطراب طيف التوحد"

// UI domain
i18n.t('ui.buttons.save') // "حفظ"
i18n.t('ui.navigation.breadcrumb') // "مسار التنقل"
i18n.t('ui.forms.validation.required') // "هذا الحقل مطلوب"

```

#### Naming Best Practices
- Use descriptive, hierarchical keys
- Avoid abbreviations
- Include context in key names
- Group related translations
- Version control translation changes

### Text Expansion Considerations

#### Arabic Text Length Factors
- Arabic text typically 20-30% longer than English
- Some terms can be 50%+ longer
- UI components must accommodate expansion
- Line height adjustments needed for Arabic

#### UI Container Sizing
```css
/* Account for Arabic text expansion */
.form-label {
  min-width: 150px; /* English baseline */
  min-width: 180px; /* Arabic expansion */
}

.button-text {
  padding: 8px 16px; /* English */
  padding: 8px 20px; /* Arabic - extra horizontal padding */
}

/* Example specific expansions */
.short-text { min-width: 120px; } /* "Save" → "حفظ" */
.medium-text { min-width: 200px; } /* "Assessment" → "تقييم شامل" */
.long-text { min-width: 300px; } /* Complex medical terms */
```

#### Component Design Guidelines
```typescript
// React component with Arabic text consideration
interface TextComponentProps {
  text: string;
  language: 'ar' | 'en';
  expandable?: boolean; // Allow text to expand
}

const TextComponent: React.FC<TextComponentProps> = ({ 
  text, 
  language, 
  expandable = true 
}) => {
  const expansionClass = language === 'ar' && expandable 
    ? 'text-expansion-ar' 
    : '';
    
  return (
    <span className={`base-text ${expansionClass}`}>
      {text}
    </span>
  );
};
```

### Input Validation for Arabic Text

#### Arabic Text Validation Patterns
```typescript
// Comprehensive Arabic text validation
const arabicTextValidation = {
  // Basic Arabic characters
  basicArabic: /^[\u0600-\u06FF\u0750-\u077F\u08A0-\u08FF\uFB50-\uFDFF\uFE70-\uFEFF\s]+$/,
  
  // Arabic with common punctuation
  arabicWithPunctuation: /^[\u0600-\u06FF\u0750-\u077F\u08A0-\u08FF\uFB50-\uFDFF\uFE70-\uFEFF\s.,!?؟]+$/,
  
  // Arabic names (letters, spaces, common name particles)
  arabicNames: /^[\u0600-\u06FF\u0750-\u077F\u08A0-\u08FF\uFB50-\uFDFF\uFE70-\uFEFF\s]+$/,
  
  // Mixed Arabic and English
  mixedArabicEnglish: /^[\u0600-\u06FF\u0750-\u077F\u08A0-\u08FF\uFB50-\uFDFF\uFE70-\uFEFF\sa-zA-Z0-9\s.,!?؟]+$/,
  
  // Arabic with numbers (both Arabic-Indic and Western)
  arabicWithNumbers: /^[\u0600-\u06FF\u0750-\u077F\u08A0-\u08FF\uFB50-\uFDFF\uFE70-\uFEFF\s0-9\u06F0-\u06F9]+$/
};

// Implementation example with Zod
import { z } from 'zod';

const arabicNameSchema = z.string()
  .min(2, 'يجب أن يكون الاسم حرفين على الأقل')
  .max(50, 'يجب أن لا يزيد الاسم عن 50 حرفاً')
  .regex(arabicTextValidation.arabicNames, 'يجب أن يحتوي الاسم على أحرف عربية فقط');
```

#### Common Validation Scenarios
```typescript
// Medical notes validation
const medicalNotesSchema = z.string()
  .min(10, 'الملاحظات الطبية يجب أن تكون 10 أحرف على الأقل')
  .max(500, 'الملاحظات الطبية يجب أن لا تزيد عن 500 حرف')
  .regex(arabicTextValidation.mixedArabicEnglish, 'يمكن استخدام العربية والإنجليزية فقط');

// Phone number validation (Saudi format)
const saudiPhoneSchema = z.string()
  .regex(/^(\+966|0)?5[0-9]{8}$/, 'رقم الهاتف غير صحيح')
  .transform(phone => phone.startsWith('+966') ? phone : '+966' + phone.replace(/^0/, ''));
```
